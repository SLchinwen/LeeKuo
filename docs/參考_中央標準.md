# 參考｜中央標準

- **用途：** 本專案撰寫文件、產出規格與程式規劃時，須參考之中央標準與引用方式。
- **適用：** 需求方、專案團隊與協作工程團隊；產出前請依對應標準檢核。

---

## 1. 本專案採用的中央標準一覽

| 標準名稱 | 適用範圍 | 引用時機 |
| --- | --- | --- |
| **Markdown 合規規範** | 所有 `.md` 文件 | 凡生成、新增或大幅修改 Markdown 時，須依該規範撰寫，以通過 markdownlint 檢核。 |
| **表單與名詞對照清單｜欄位引用標準** | 表單、API、資料表欄位 | 新增或修改表單／API／欄位時，欄位命名與名詞須與該清單對齊，並同步更新清單。 |
| **程式編碼原則｜共用規範** | 程式代號、TABLE、API、雛型命名 | 主程式代號、子編碼、批次與雛型檔名須依該規範編列。 |
| **標準分析產出要求**（治理規範 §6） | PRD、交付包、功能作業產出 | 各功能作業 PRD 或交付包須含：模組與主程式代碼、權限矩陣、情境說明、欄位中英文對照表。 |

上述標準文件存放於專案倉儲之範本庫、規格目錄與內部治理目錄；撰寫或產出時請於專案內查閱最新版。

---

## 2. 如何設定「參考中央標準」

- **撰寫 Markdown：** 生成或大幅修改 `.md` 前，開啟並遵守 `00_Templates/Markdown_合規規範.md`（清單用 `-`、標題用 `##`／`###`、表格管線右側留空等）。
- **表單／API／欄位：** 命名與用途對齊 `03_Solution/表單與名詞對照清單_欄位引用標準.md`；新增欄位時同步更新該清單。
- **程式代號與雛型：** 主程式代號與雛型檔名依 `internal_docs/程式編碼原則_共用規範.md`；雛型檔名建議 `YYYY-MM-DD_prototype_<程式代號>.html`。
- **交付包與 PRD：** 依 `internal_docs/docs_治理規範.md` §6 與協作與治理規範 §2.4，產出四項必備（模組與主程式代碼、權限矩陣、情境說明、欄位對照表）。
