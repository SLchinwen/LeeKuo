# 雛型轉線上動態操作模擬說明｜建議做法

本文件說明如何將既有**雛型（HTML）**與 **PRD／操作情境**轉成**線上的動態操作模擬說明**，供訓練、操作手冊或模擬練習使用；並區分**AI 可產出**與**需搭配其他工具**的部分。

---

## 1. 目標與產物

| 目標 | 產物範例 |
| :--- | :--- |
| **操作說明文件** | 依程式代號分章，每支程式有「目的、操作步驟、畫面說明、對應雛型連結」之 Markdown 或網頁。 |
| **線上動態操作模擬** | 一頁式或多頁式網站：左側（或上方）為**步驟說明**，右側（或下方）為**雛型畫面**（iframe 或連結）；學員可依步驟在雛型上對照操作。 |
| **引導式操作教學**（進階） | 在雛型畫面上疊加「步驟 N：請點選○○按鈕」之提示，點擊下一步時高亮對應區塊（需 JS 套件或工具）。 |

---

## 2. 建議做法總覽

| 階段 | 做法 | 誰來做／用什麼 |
| :--- | :--- | :--- |
| **① 結構化操作步驟** | 從 PRD、情境說明、雛型畫面整理出「程式代號 → 操作步驟清單（步驟 1、2、3…＋說明＋對應畫面區塊）」 | **AI 可產出**：依 PRD 與雛型結構產生 Markdown 或 JSON／YAML 步驟檔 |
| **② 訓練入口頁（步驟＋雛型同屏）** | 單一 HTML：左側步驟列表、右側 iframe 嵌雛型；點選步驟可捲動或切換說明，雛型維持可操作 | **AI 可產出**：靜態 HTML＋少量 JS，可直接使用 |
| **③ 操作手冊網頁版** | MkDocs 或靜態站：每支程式一頁，內含步驟與雛型連結（或嵌 iframe） | **AI 可產出**：Markdown 內容；MkDocs 需專案既有建置 |
| **④ 畫面上引導（高亮、下一步）** | 在雛型頁面上疊加「步驟泡泡」、點下一步高亮表單或按鈕 | **需其他工具**：Shepherd.js、Intro.js、或自寫 JS；**AI 可產出**步驟文案與選擇器設定檔 |
| **⑤ 影片／錄影教學** | 錄製實際操作流程，搭配旁白 | **需其他工具**：OBS、Loom、螢幕錄影＋剪輯 |
| **⑥ 正式環境模擬（含後端）** | 訓練用環境，可真實登入、存讀假資料 | **需開發**：後端／資料庫／權限隔離，非雛型可涵蓋 |

---

## 3. AI 能產出的內容（本專案可直接請 AI 做）

以下皆可由 **Cursor／AI** 依既有雛型與 PRD 產出，無需額外付費工具：

1. **操作步驟結構（Markdown 或資料檔）**  
   - 依程式代號產出「目的、前置條件、操作步驟 1～N、注意事項」；可對應雛型上的區塊或按鈕名稱。  
   - 產物：`02_Analysis/YYYY-MM-DD_操作步驟_程式代號.md` 或集中於一檔。

2. **訓練入口頁 HTML（步驟＋雛型同屏）**  
   - 單一 HTML 檔：左側（或可收合）為步驟列表，右側為 iframe 載入雛型；點選步驟時右側說明區更新，雛型維持可點選、可填表。  
   - 產物：如 `docs/prototype/training_hub_CUS110.html`（見本專案範例）。  
   - 部署：與現有雛型同站（如 GitHub Pages），無需後端。

3. **操作手冊章節（Markdown）**  
   - 每支程式一節：標題、目的、步驟、欄位說明、雛型連結；可納入 `docs/` 或 MkDocs 手冊。

4. **引導式教學的「步驟設定」**  
   - 若日後採用 Shepherd.js／Intro.js：AI 可產出「步驟順序、說明文案、對應 CSS 選擇器或 id」之設定檔（JSON／JS），由專案匯入套件後套用。

**限制**：AI 產出為靜態 HTML／Markdown／設定檔；**不包含**錄影、語音、或需登入之後端模擬環境。

## 4. 需搭配的其他工具（簡表）

| 需求 | 建議工具 | 說明 |
| :--- | :--- | :--- |
| **畫面上逐步高亮／引導** | Shepherd.js、Intro.js、Driver.js | 在既有雛型頁面掛上 JS，顯示「步驟 N：請點選○○」泡泡；AI 可產步驟內容與選擇器，套件需自行引入。 |
| **錄影＋旁白** | OBS、Loom、QuickTime（Mac） | 錄製操作流程，上傳至影片平台或內嵌於手冊。 |
| **SCORM／LMS 整合** | iSpring、Articulate、或 LMS 平台 | 若需匯入企業學習管理系統，需轉成 SCORM 或該平台格式。 |
| **正式環境之訓練站** | 開發團隊建置 | 獨立訓練用環境、假資料、權限隔離，非雛型範圍。 |

---

## 5. 建議實作順序（本專案）

1. **先做**：請 AI 產出 **① 操作步驟 Markdown** 與 **② 訓練入口頁 HTML**（步驟＋iframe 雛型），放置於 `docs/prototype/` 或約定目錄，與現有雛型一併發布。學員開啟訓練入口頁即可「左看步驟、右操作雛型」。
2. **可選**：將操作步驟整理進 **操作手冊**（`docs/` 或 MkDocs），每支程式一頁，並附雛型連結或嵌 iframe。
3. **進階**：若需「畫面上點下一步就高亮」：在雛型 HTML 或複本中引入 Shepherd.js／Intro.js，再請 AI 產出該程式的步驟設定檔（文案＋選擇器），由負責人貼入專案即可。

---

## 6. 範例與放置位置

- **訓練入口頁範例**：本專案已提供 `docs/prototype/training_hub_example.html`，左側為步驟列表、右側為雛型 iframe，點選步驟可更新說明文字；可依程式代號複製改寫步驟與 iframe 的 `src`。  
- **操作步驟範例**：可置於 `02_Analysis/` 或 `docs/manuals/`，檔名如 `YYYY-MM-DD_操作步驟_CUS100.md`。

雛型若已符合 [文件治理規範](docs_治理規範.md) §7（操作一致性、響應式、欄位說明一致、可轉操作說明與訓練來源），轉成上述產物時會較順暢。
