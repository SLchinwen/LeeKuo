# 本專案目錄對應說明｜立國專案

- **版本：** v1.0
- **日期：** 2026-02-26
- **用途：** 治理規範概念與本專案（LeeKuo）實際路徑對照，供產出或搬移文件時判斷放置位置。

---

## 對照表

| 治理規範概念 | 本專案實際路徑 | 備註 |
| --- | --- | --- |
| **對外文件（使用者／客戶）** | `docs/` | MkDocs 建置來源；含首頁、專案管理服務總覽、操作手冊與 SOP、待決議事項、任務與甘特圖；MCR/HR 子目錄為操作手冊與 SOP 分階 |
| **分析／需求整理** | `02_Analysis/` | 需求整理、情境彙整、流程圖、變更對照、分析建議 |
| **PRD／規格／移交** | `03_Solution/` | PRD、工程規格草案、移交摘要、表單對照、簡報、專案進行進度、泳道圖 |
| **雛型** | `03_Solution/` 開發用；**對外**須同步放 `docs/prototype/` | 操作手冊之「對應畫面」以雛型連結取代靜態圖片；雛型檔名建議 `YYYY-MM-DD_prototype_<程式代號>.html` |
| **內部治理** | `internal_docs/` | 協作與治理、文件治理、程式編碼原則、本說明；不納入 MkDocs |
| **範本與合規** | `00_Templates/` | 範本、Markdown_合規規範、立國專案_專案管理與文件治理規則 |
| **追溯／歸檔** | `99_Trace/` | 已歸檔之需求、情境、版本紀錄 |
| **診斷期產物** | `01_Discovery/` | 訪談、背景、診斷期產物 |

---

## 補充

- **導覽標題**：`mkdocs.yml` 之 `nav` 標題建議 ≤8 個中文字。
- **人員呈現**：對外文件（含 docs）一律以**職務稱謂**呈現，不寫個人姓名。
- **日期**：專案管理文件之「更新日」「日期」須與實際日期一致；產出時以系統日期填入。
