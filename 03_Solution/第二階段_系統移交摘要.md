# 第二階段_系統移交摘要
- **日期：** 2026-01-29
- **用途：** 給工程師的移交摘要（需求、流程、關鍵決策）

## 1. 需求摘要
- 回報與拍照**雙軌**（need_report / need_photo）
- 審核支援**退回補件**
- 照片**補拍/換圖**保留歷史
- 文件產出可依條件**過濾與追溯**
- 吊車欄位由主檔維護，**回報端只讀**
- **設備完工回報**（通知可請款，可重開修改）

## 2. 關鍵流程
請參考 `02_Analysis/第二階段_流程圖.md`

## 3. 關鍵欄位
- need_report / need_photo
- photo_status（Pending / Completed / NotRequired）
- crane_tonnage / parking_zone_group

## 4. 決策與限制（已確認）
- **關帳後資料不可再變動**
- **專案經理送出請款文件後方可關帳**
- **關帳定義為合約關帳（請款關帳後不允許更正）**
- 文件產出為快照，需保留 metadata（產出時間、條件）
- **PhotoPending 可核定/請款**
- **退回後可重新回報**

## 5. 驗收重點
- need_photo=否 → 回報不可阻擋
- need_photo=是 → 回報完成且 photo_status=Pending
- 文件產出可追溯篩選條件
- 設備完工回報可重開修改
