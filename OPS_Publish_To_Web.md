# AI 網站發布執行手冊

## 目的
將 `03_Solution` 或 `02_Analysis` 中的定稿文件，清洗敏感資訊後，發布至 `docs/` 資料夾以供 MkDocs 生成網站。

## 執行邏輯
1.  **來源選擇：** 等待使用者指定要發布的原始文件 (例如 `@PRD.md`)。
2.  **清洗與轉換 (Sanitize & Transform):**
    * 移除所有「內部備註」、「成本分析」章節。
    * 移除 YAML 檔頭中的 `owner`, `status` 等內部欄位。
    * 確保 Mermaid 圖表語法正確。
    * 將標題層級 (H1, H2) 調整為適合網頁閱讀的結構。
3.  **歸檔至 docs：**
    * 移動到 `docs/` 對應的子目錄 (例如 `docs/manual/`).
    * 如果 `docs/index.md` (首頁) 不存在，請幫我生成一個目錄頁。
