# Markdown 合規規範

- **用途：** 避免 markdownlint 警告與格式不一致
- **適用：** 所有 `.md` 文件（含 AI 生成文件）

## 1. 無序清單（MD004）

- **一律使用破折號 `-` 作為無序清單符號，不要使用星號 `*`。**
- 同一份文件內清單樣式須一致；本專案預設為 `-`。

```markdown
<!-- 正確 -->
- 項目一
- 項目二

<!-- 避免 -->
* 項目一
* 項目二
```

## 2. 標題與小節（MD036）

- **若該行是「小節標題」，請用正式標題語法（`##` / `###` / `####`），不要單獨用粗體當標題。**
- 「粗體＋冒號」可作為標籤（如 **目的：**、**理由：**），但若其後緊接多個清單項，建議在標籤後加一空行再列清單，避免被 linter 判為「強調當標題」。

```markdown
<!-- 正確：小節用標題 -->
#### 原規劃
- 內容…

#### 調整後
- 內容…

<!-- 避免：單行粗體當標題 -->
**原規劃**
* 內容…
```

## 3. 標題與清單空白

- 標題前後需保留一行空白（對應 MD022）
- 清單前後需保留一行空白（對應 MD032）

## 4. 表格格式

- 表格欄位兩側需保留空白（對應 MD060）
- 正確格式範例：

| 欄位 | 說明 |
| --- | --- |
| A | B |

## 5. 其他建議

- 盡量使用 ASCII
- 避免多餘空行與尾端空白
- 避免重複標題文字（對應 MD024）

## 6. AI 生成文件檢查清單

撰寫或生成 `.md` 時請自檢：

1. 無序清單是否全部用 `-`（未用 `*`）？
2. 小節標題是否用 `##` / `###` / `####`，而非單行 **粗體**？
3. 「標籤＋清單」時，標籤行後是否有一空行再接清單？
